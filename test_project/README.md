# test_project

Test environment for the Argon fork, decomposed from a Roblox place with the
offline **`argon syncback`** translation layer.

## Layout

```
test_project/
├── default.project.json   # project: DataModel + 13 services, each $path -> src/<Service>
├── argon.toml             # project-local config (overrides global ~/.argon/config.toml)
├── src/                   # source tree (one folder/file per instance)
├── template/template.rbxl # reference place
├── test.rbxl              # original place this src/ was decomposed from
└── place.rbxl             # rebuilt place (generated by `argon build`)
```

There is **no per-project `.argon/` directory**. Argon keeps no on-disk index;
it rebuilds the instance tree from `src/` live on every command. The only
`.argon` is the global `~/.argon/` (templates, global config) — and this project
overrides the relevant parts via `argon.toml`.

`<argon>` below = the fork binary: `../argon/target/debug/argon`.

## Regenerate the place (src/ -> rbxl)

Build the source tree back into a Roblox place file:

```bash
<argon> build default.project.json -o place.rbxl       # binary .rbxl
<argon> build default.project.json -o place.rbxlx --xml # XML .rbxlx
```

Do not target `test.rbxl` — that is the original input. Default output is
`<name>.rbxl` (here `test.rbxl`), so always pass `-o` to avoid clobbering it.

Open `place.rbxl` in Studio, or `--watch` to rebuild on every file change:

```bash
<argon> build default.project.json -o place.rbxl --watch
```

## Regenerate the source (rbxl -> src/)

Offline decompose a place/model back into `src/` (reverse of build):

```bash
<argon> syncback test.rbxl default.project.json          # add/update in place
<argon> syncback test.rbxl default.project.json --prune   # also remove instances gone from the place
```

`syncback` diffs the place against the current tree: new instances are written,
changed properties updated, and (with `--prune`) instances missing from the
place are removed. Accepts `.rbxl`, `.rbxlx`, `.rbxm`, `.rbxmx`.

## Live two-way sync

```bash
<argon> serve default.project.json   # then connect from the Argon Studio plugin
```

## Duplicate-name siblings

`argon.toml` sets `keep_duplicates = true`. Sibling instances sharing a name are
written as `Name_<uuid>/` folders, with the real name stored as `originalName`
in `init.meta.json`. `build` reads `originalName` and restores the true name in
the place, so duplicates round-trip.

With `keep_duplicates = false` (Argon's default) the colliding siblings are
skipped with an `already exists! Skipping` warning instead.

## Config

`argon.toml` here overrides the global config for any command run from this
directory (a workspace config fully replaces the global one). It sets only the
keys that differ from Argon defaults; see `argon/src/config.rs` for the full
list. Inspect / change values with:

```bash
<argon> config                       # list
<argon> config <key> <value>         # set
<argon> config <key>                 # reset <key> to default
```
